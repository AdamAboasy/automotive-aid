## نظرة عامة

التحويل من ملف HTML/JS واحد يعتمد على localStorage إلى تطبيق TanStack Start كامل مع قاعدة بيانات Lovable Cloud وأدوار مستخدمين. النطاق ضخم، فهنقسمه على مراحل واضحة وكل مرحلة تخرج بنسخة شغّالة.

---

## المرحلة 1: الأساس (هنبدأ بيها دلوقتي)

**الهدف:** هيكل المشروع + تسجيل الدخول + الأدوار + شكل الواجهة الأساسي.

- تفعيل Lovable Cloud
- نظام Auth (إيميل + كلمة سر) — صفحة `/auth`
- جدول `profiles` + جدول `user_roles` بـ enum:
  `admin`, `reception`, `workshop_manager`, `technician`, `hr`
- دالة `has_role()` و RLS مبدئية
- الـ Layout الأساسي بالعربي/RTL، خط Cairo، Tab Navigation زي الأصلي
- 4 routes فاضية: `/reception`, `/workshop`, `/hr`, `/settings`
- حماية الـ routes حسب الدور
- صفحة "غير مصرح" للأدوار اللي ماتقدرش تدخل تبويب معين

**النتيجة:** المستخدم يقدر يسجل، يدخل، ويشوف التبويبات حسب صلاحيته. لسه مفيش بيانات حقيقية.

---

## المرحلة 2: الإعدادات والبيانات الأساسية

**الهدف:** كل القوائم المرجعية في قاعدة البيانات.

- جداول: `banks`, `body_types`, `brands` (مع `models` و `engines`)
- جداول: `warehouses`, `workshops`, `employees`
- تبويب الإعدادات شغّال: إضافة/تعديل/حذف
- Seed بالبيانات الافتراضية اللي في `kia.js`

---

## المرحلة 3: قسم الاستقبال

- جدول `clients` + شاشة العملاء (بحث، إضافة، تعديل، عرض السيارات)
- جدول `cars` (مربوط بالعميل)
- جدول `inventory` (المخزون داخل المخازن)
- جدول `maintenance_bookings` (حجوزات الصيانة)
- جدول `complaints` (الشكاوى) — **جديد، مش موجود في الأصل**
- جدول `client_followups` (متابعة العملاء) — **جديد**

- شاشات الفنيين، حالة الأوامر، تتبع
- ربط الأوامر بالعميل والسيارة والورشة

## المرحلة 6: تحسينات

---

## تفاصيل تقنية

- **Stack:** TanStack Start (موجود) + Supabase عبر Lovable Cloud + Tailwind + shadcn/ui
- **اللغة:** عربي RTL، خط Cairo، الثيم Dark كما في الأصل (هنحط الألوان في design tokens بدل ما-hardcoded)
- **الأدوار:** جدول منفصل `user_roles` + دالة `has_role()` مع `SECURITY DEFINER` (لتفادي الـ recursion)
- **RLS:** كل جدول هيكون عليه RLS من أول يوم، مع GRANTs صريحة
- **الـ Server functions:** القراءة والكتابة عبر `createServerFn` مع `requireSupabaseAuth`
- **بيانات قديمة:** هنوفر زرار "استيراد من localStorage" في المرحلة 6 عشان أي بيانات حالية ماتضيعش

## اللي هينفّذ في الرد القادم لو وافقت

**المرحلة 1 كاملة فقط:**
1. تفعيل Lovable Cloud
2. إنشاء جداول `profiles`, `user_roles`, الـ enum، الـ trigger، RLS، GRANTs
3. صفحة `/auth` (تسجيل دخول + تسجيل جديد)
4. الـ Layout الرئيسي بالـ navigation
5. الأربع routes حسب الدور
6. صفحة Settings فيها فقط "إدارة المستخدمين والأدوار" (للأدمن)

بعد كده نتحرك للمرحلة 2 ونكمل واحدة واحدة. كل مرحلة قابلة للاختبار لوحدها.
